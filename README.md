# Trip Photo Archive (Web Edition)

一个专注于"行程-事件-照片"三层结构的智能旅游记忆管理系统。受 Notion 启发，采用极致的交互动效与本地优先的数据管理理念。

- **全新 UI 设计**：
  - **Premium 视觉**：采用深色背景 (#0b0c10) 与极致毛玻璃特效，分类与城市标签自带背景模糊与光效。
  - **响应式横屏网格**：照片墙固定 **10 列**，照片卡片宽度充分利用屏幕空间。
  - **智能过滤器 (Filter Menu)**：右上角集成多选过滤器，支持"未分类"、"未标记城市"等组合逻辑。
  - **底部总控栏 (ActionBar)**：选中照片时自动滑出，支持批量合并、属性修改与删除。
- **全系管理系统**：
  - **多语言支持 (i18n)**：完整的中文 (简体) 与英文双语界面，支持一键切换。
  - **自定义元数据**：支持全局分类、城市与标签的增删改查。
  - **三层架构**：行程 (Trip) -> 事件 (Event) -> 照片 (Photo) 的层级化管理。
  - **封面系统**：可为行程和事件一键设置封面。
- **极致交互**：
  - **智能右键菜单**：右键点击即自动选中，支持边界翻转检测，快速创建新城市或批量评分。
  - **物理级动效**：基于 **Framer Motion** 的平滑过滤、卡片堆叠感与无缝场景转换。
  - **沉浸式灯箱**：带元数据侧边栏的同步浏览灯箱。
- **本地优先 (Local-First)**：直接读写本地 `trip_database.json`，三重去重机制确保 Windows 路径兼容性。
- **高性能**：
  - **IDB 缩略图缓存**：首次扫描后生成 600px 缩略图存入 IndexedDB，滚动时直接读取，不加载原图。
  - **真正的虚拟滚动**：基于 `@tanstack/react-virtual`，DOM 中始终只保留 ~50 个卡片，千张照片库流畅无卡顿。

## 🛠️ 技术栈

- **前端核心**：React 19, Vite 8
- **动效库**：Framer Motion 12
- **图标集**：Lucide React
- **虚拟渲染**：@tanstack/react-virtual
- **样式系统**：Tailwind CSS 4
- **持久化**：本地 JSON + File System Access API + IndexedDB（缩略图缓存）

## 📦 安装与运行

1. 确保已安装 Node.js 环境。
2. 克隆项目后，在根目录运行：
   ```bash
   npm install
   ```
3. 启动开发服务器：
   ```bash
   npm run dev
   ```
4. 在浏览器中打开提示的地址（推荐使用 Chrome 或 Edge）。

## 📖 使用指南

1. **初始化**：首次启动点击"选择文件夹"，授权访问您的照片存放目录。系统会自动扫描照片、读取 EXIF、生成缩略图。
2. **浏览**：单选照片可进行批量操作，双击可进入沉浸式灯箱浏览。
3. **归档**：
   - 右键照片 -> 聚集到新事件。
   - 右键事件 -> 归档到已有行程（层级菜单）或创建新行程。
4. **属性管理**：在灯箱侧边栏或创建弹窗中，可以为事件设置地点、评分、笔记等。
